Refine an existing image — either generated by this tool or any PNG/JPG you point at. Use the image-refinement skill for the workflow.

Two flows:
1. **Tweak the prompt and regenerate fresh** — best for direction changes (different lighting, medium, angle, mood). Produces a new image.
2. **Image-to-image edit** — best for surgical fixes (warmer light, remove an element, color correction). Keeps the same image, edits in place.

The skill asks which one you want, then dispatches.

## Usage

- `/refine <path>` — interactive. Reads the image's sidecar at `<dir>/.meta/<basename>.json` (if present), shows you the original prompt and provider, asks tweak-or-edit.
- `/refine <path> tweak` — pre-pick mode 1 (prompt tweak + regenerate).
- `/refine <path> edit` — pre-pick mode 2 (image-to-image edit).
- `/refine <path> --provider <name>` — override the provider (`gemini` or `openai`). Defaults to whichever the original image was generated with (from the sidecar).
- `/refine <path> --quality <q>` — OpenAI only. Override quality tier on the refinement.
- `/refine <path> --out <new-path>` — override save location. Default avoids overwriting the original.

Examples:

```
/refine ./generated/_images/fisherman-on-a-20260502-143055.png
/refine ./blog-header.png edit
/refine ./hero.png tweak
```

If you point at a foreign image (no sidecar), only image-to-image mode is available — there's no original prompt to tweak. The skill will tell you and proceed.
